Getting Started

Poke Aug 23, 2018

All Poke’s components are hosted on GitHub under the warp-poke organisation.

This organisation provides some agents in order to check protocols:

  • HTTP: https://github.com/warp-poke/http-rust-agent
  • DNS: https://github.com/warp-poke/dns-rust-agent
  • SSL: https://github.com/warp-poke/ssl-go-agent

Furthermore, there is an API that manage checks and their scheduling:

  • API: https://github.com/warp-poke/poke-api.
Start using docker-compose

On the poke-compose repository, there is all you need to setup a docker-based poke infrastructure. This infrastructure used the standalone version of each distributed components.

Firstly, get the repository using git:

$ git clone https://github.com/warp-poke/poke-compose.git

Once, the repository is cloned, start the infrastructure using the docker-compose command:

$ docker-compose --project-name poke up
